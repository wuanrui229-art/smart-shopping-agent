# 原项目整合说明

## 可复用内容盘点

原压缩包是一套按周提交的购物 Agent 项目，不是一个可以直接运行的统一工程。整合时保留了其中最有价值的设计和算法思想：

| 原材料 | 已整合到新系统 |
|---|---|
| Week 1 评论情感与风险分析 | 评论可信度、情感比例、风险标签 |
| Week 2 商品检索与联调 | 演示商品目录、预算过滤、候选商品生成 |
| Week 3/4 决策 Agent | 需求解析→检索→评论分析→排序的状态化流程，以及防死循环思路 |
| Week 5 记忆模块 | SQLite 会话历史和用户偏好 |
| Week 6/7 Streamlit/HTML 原型 | 聊天、历史、设置、六区决策面板的交互思路 |
| PRD v1.2 | 1 主推 + 2 备选、评分权重、授权下单、澄清规则、MVP 边界 |

## 主要补齐内容

原项目最后的 Streamlit 界面已经预留 FastAPI 地址，但压缩包中没有真正的 FastAPI 服务，且大部分结果来自前端 `DEMO_DATA`。新版本完成了以下补齐：

1. 新建可运行的 FastAPI 后端与 REST API。
2. 将前端、后端和算法拆分成独立目录。
3. 把 Week 1–5 的分散逻辑重组为一条可执行算法流水线。
4. 增加 SQLite 会话、偏好与授权记录。
5. 增加可选的 OpenAI Responses API 结构化解析。
6. 增加没有密钥时的稳定降级路径。
7. 增加自动化测试、接口文档和一键启动脚本。
8. 重新设计前端，使评分公式、评论风险与推荐理由可以直接展示给老师。

## 课程版边界

由于原压缩包没有 Amazon 正式 API 授权、评论数据授权和支付授权，新系统使用内置演示目录，不进行真实抓取或真实下单。这不是功能缺失，而是课程演示中更安全、可复现的实现。以后获得合规接口与密钥后，可替换算法层的数据源而不修改前端结构。
