# Smart Shopping AI 系统设计与答辩说明

## 1. 作业定位

Smart Shopping AI 是一套面向“选择困难、评论不可信、信息过载”问题的自主决策型购物助手。当前前端恢复自 Week 7 原版成品，并扩展成开放式多轮购物对话：任何消费品类都能交流和生成 Top 3 建议；Kids Backpack、Headphones、Running Shoes、Tablet、Keyboard 和 Smart Watch 六类另有课程演示数据和确定性评分。

## 2. 三层架构

```text
浏览器前端
  ↓ REST/JSON + NDJSON 实时事件流
FastAPI 后端 ── SQLite（会话、偏好、授权记录）
  ↓ 调用
算法层 ── LLM 对话与意图路由 / 开放品类生成 / 评论可信度 / 确定性排序
```

前端只负责展示和交互，不再保存原版静态推荐数据；后端负责数据校验、状态、偏好、模型访问和接口；算法层独立完成对话意图识别、开放品类候选生成、六品类路由、偏好感知多维排序，以及评论可信度决策管线。因此三个部分边界清晰，符合“前端、后端、算法”三部分要求。

## 3. 大语言模型在哪里发挥作用

线上版本通过 Vercel AI Gateway 调用 `openai/gpt-5.4-mini`。模型读取最近对话并输出受 JSON Schema 约束的结构化结果，包括：聊天意图、是否需要追问、品类、预算、关注点和三款候选，因此“你好”“继续比较第二款”和任意商品需求不再被同一个固定问句处理。

系统采用两条清晰的数据路径。六个内置品类中，商品事实来自课程数据集，大模型只负责理解和路由，最终排序由确定性公式完成。其他品类由大模型生成通用选购建议和适配度估算，页面会醒目标注“AI-generated guidance”，不声称使用了实时价格、库存、评分或评论。模型不可用时，六个内置品类仍能离线演示。

## 4. 评论可信度算法

每个商品的可信度分数由四项组成：

- 已验证购买占比：35%
- 非重复评论占比：25%
- 具有细节的评论占比：20%
- 评论时间分散度：20%

系统同时统计正面、中性、负面情感，识别重复文案、低验证率、发布时间集中和低分评论过多等风险。可信度 70–100 为低风险，40–69 为中风险，0–39 为高风险。

## 5. 推荐决策算法

```text
综合分 = 评论可信度 × 0.40
       + 需求匹配度 × 0.30
       + 价格竞争力 × 0.20
       + 平台评分 × 0.10
```

选择评论可信度作为最高权重，是因为前期 Reddit 用户研究显示“不信任平台评论”是最高频痛点。前端同步显示每一项分数，因此推荐结论可审计、可解释。

## 6. 安全与合规设计

- 输入最大 500 字符，后端使用 Pydantic 做类型与长度校验。
- 前端文本在插入页面前进行转义，降低脚本注入风险。
- 大模型密钥仅从环境变量读取，前端永远拿不到密钥。
- 任何加购动作都必须传入 `authorized=true`，否则后端拒绝。
- 课程版只记录演示授权，不连接真实支付或创建真实订单。
- 用户偏好使用本机 SQLite 保存，不上传第三方平台。

## 7. 实时交互设计

原版页面默认调用 `POST /api/original/chat/stream`。后端在一次请求中依次返回接收请求、读取偏好、识别品类、加载评论统计、评分排序和最终结果事件。前端边接收边把它们映射为“正在理解需求”“正在匹配类别”“正在生成推荐”等单行状态；结果或追问返回后自动隐藏，不向普通用户暴露请求编号、耗时和内部流水线细节。

如果部署平台或旧版本不支持流式接口，前端只在收到 404/405 时降级调用 `POST /api/original/chat`，避免对同一需求重复提交。请求处理中输入框、发送按钮和快捷入口会暂时禁用，防止重复点击。

“实时”在本课程版本中指前端、后端和算法在用户提交后即时通信并重新计算，不代表实时抓取 Amazon 价格或执行真实下单。六类商品和评论统计来自课程演示数据集；其他品类是大模型知识范围内的估算建议，购买前必须核验。

公开演示优先使用 Vercel：`https://smart-shopping-agent-wuanrui.vercel.app`。Vercel 将 FastAPI 应用部署为 Python Function，并支持当前 NDJSON 流式响应。由于 Serverless 本地磁盘是临时的，前端会在每次推荐请求中携带当前用户偏好；SQLite 在 Vercel 上只承担临时会话缓存，不作为永久数据库。若后续需要多用户长期保存，应接入 Postgres 等托管数据库。

## 8. 答辩演示路线（6 分钟）

1. 打开首页，说明这是 Week 7 原版白底橙色界面，而非重新设计的替代页面。
2. 先输入“你好”，再输入一个六类之外且条件完整的需求，展示开放对话与任意品类推荐。
3. 指出页面对开放品类使用估算标签，不伪造实时商品数据。
4. 输入“我想买一双适合日常训练的跑鞋”，展示内置数据和 Top 3 深度商品卡。
5. 解释四维评分、评论情感、优势与风险信息。
6. 修改价格敏感度后重复跑鞋需求，展示排序变化。
7. 打开 `/docs`，展示 FastAPI 自动生成的后端接口文档。

逐句计时讲稿和现场备用方案见 `docs/六分钟作业展示讲稿.md`。

## 9. 可继续扩展的部分

- 将演示商品目录替换为 Amazon Creators API 或合规商品 API。
- 使用真实人工标注评论集评估可信度算法准确率。
- 接入向量数据库做更大规模的语义检索。
- 将演示加购替换为获得正式授权的商城接口。
- 增加管理员评测面板，记录推荐采纳率和用户满意度。
