# Smart Shopping AI 系统设计与答辩说明

## 1. 作业定位

Smart Shopping AI 是一套面向“选择困难、评论不可信、信息过载”问题的自主决策型购物助手。当前前端恢复自 Week 7 原版成品，支持 Kids Backpack、Headphones、Running Shoes、Tablet、Keyboard 和 Smart Watch 六类演示，并由后端返回明确的 Top 3 推荐。

## 2. 三层架构

```text
浏览器前端
  ↓ REST/JSON + NDJSON 实时事件流
FastAPI 后端 ── SQLite（会话、偏好、授权记录）
  ↓ 调用
算法层 ── LLM 解析 / 规则降级 / 评论可信度 / 加权排序
```

前端只负责展示和交互，不再保存原版静态推荐数据；后端负责数据校验、状态、偏好和接口；算法层独立完成六品类识别、偏好感知多维排序，以及原有的评论可信度决策管线。因此三个部分边界清晰，符合“前端、后端、算法”三部分要求。

## 3. 大语言模型在哪里发挥作用

配置 `OPENAI_API_KEY` 后，系统使用 OpenAI Responses API 将复杂自然语言解析为结构化 JSON，包括：品类、预算、功能需求、品牌偏好、避坑项、使用场景和置信度。

大模型不直接决定商品分数。所有价格、评论和评分均来自后端数据，最终排序由确定性公式完成，避免模型幻觉改变事实。没有密钥或模型失败时，规则引擎接管解析，使演示仍可继续。

## 4. 评论可信度算法

每个商品的可信度分数由四项组成：

- 已验证购买占比：35%
- 非重复评论占比：25%
- 具有细节的评论占比：20%
- 评论时间分散度：20%

系统同时统计正面、中性、负面情感，识别重复文案、低验证率、发布时间集中和低分评论过多等风险。可信度 70–100 为低风险，40–69 为中风险，0–39 为高风险。

## 5. 推荐决策算法

```text
综合分 = 评论可信度 × 0.40
       + 需求匹配度 × 0.30
       + 价格竞争力 × 0.20
       + 平台评分 × 0.10
```

选择评论可信度作为最高权重，是因为前期 Reddit 用户研究显示“不信任平台评论”是最高频痛点。前端同步显示每一项分数，因此推荐结论可审计、可解释。

## 6. 安全与合规设计

- 输入最大 500 字符，后端使用 Pydantic 做类型与长度校验。
- 前端文本在插入页面前进行转义，降低脚本注入风险。
- 大模型密钥仅从环境变量读取，前端永远拿不到密钥。
- 任何加购动作都必须传入 `authorized=true`，否则后端拒绝。
- 课程版只记录演示授权，不连接真实支付或创建真实订单。
- 用户偏好使用本机 SQLite 保存，不上传第三方平台。

## 7. 实时交互设计

原版页面默认调用 `POST /api/original/chat/stream`。后端在一次请求中依次返回接收请求、读取偏好、识别品类、加载评论统计、评分排序和最终结果事件，前端边接收边显示处理状态。每次请求都有独立编号和后端耗时，便于现场证明页面并非播放预设结果。

如果部署平台或旧版本不支持流式接口，前端只在收到 404/405 时降级调用 `POST /api/original/chat`，避免对同一需求重复提交。请求处理中输入框、发送按钮和快捷入口会暂时禁用，防止重复点击。

“实时”在本课程版本中指前端、后端和算法在用户提交后即时通信并重新计算，不代表实时抓取 Amazon 价格或执行真实下单。商品和评论统计来自课程演示数据集。

## 8. 答辩演示路线（6 分钟）

1. 打开首页，说明这是 Week 7 原版白底橙色界面，而非重新设计的替代页面。
2. 输入“我想买一双适合日常训练的跑鞋”，展示前端向 `/api/original/chat/stream` 发起请求，以及页面实时出现的后端处理步骤。
3. 展示需求与用户画像、市场概览、Top 3 深度商品卡和请求编号。
4. 解释四维评分、评论情感、优势与风险信息。
5. 展示多维横向比较、个性化分析与最终建议。
6. 在偏好设置中把价格敏感度从 50 调到 100，保存后再次输入完全相同的跑鞋需求，展示 ASICS 与 HOKA 的顺序变化。
7. 打开 `/docs`，展示 FastAPI 自动生成的后端接口文档。

逐句计时讲稿和现场备用方案见 `docs/六分钟作业展示讲稿.md`。

## 9. 可继续扩展的部分

- 将演示商品目录替换为 Amazon Creators API 或合规商品 API。
- 使用真实人工标注评论集评估可信度算法准确率。
- 接入向量数据库做更大规模的语义检索。
- 将演示加购替换为获得正式授权的商城接口。
- 增加管理员评测面板，记录推荐采纳率和用户满意度。
